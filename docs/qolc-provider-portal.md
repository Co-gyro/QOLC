# QOLC サービス提供者ポータル設計書

> 加盟店（サービス提供者）向けの明細アップロード・管理画面

---

## 1. 概要

### 対象ユーザー

サービス提供者（加盟店）の担当者
- 訪問診療クリニック
- 訪問看護ステーション
- 訪問歯科
- 訪問調剤薬局
- タクシー会社
- 買い物代行
- その他サービス

※介護施設も加盟店だが、施設ポータルを使用

### 主な機能

| 機能 | 説明 |
|------|------|
| 明細アップロード | CSV/Excelで利用明細（レセプト）をアップロード |
| 自動変換 | 登録済みフォーマットでQOLC統一形式に変換 |
| 決済状況確認 | アップロードした明細の決済状況を確認 |
| 履歴管理 | 過去のアップロード履歴を確認 |

---

## 2. 画面構成


## 3. 各画面の詳細

### 3.1 ダッシュボード

**目的**: 現在の状況を一目で把握

**表示内容**:

【統計カード（上部横並び）】
| カード | 表示内容 | 例 |
|--------|---------|-----|
| 今月のアップロード数 | 今月アップロードした明細の件数 | 127件 |
| 決済完了額 | 今月の決済完了金額 | ¥3,456,780 |
| 未決済件数 | 決済待ち・エラーの件数 | 3件 |
| 振込予定額 | 次回振込予定の金額 | ¥3,234,500 |

【アラート（統計カードの下）】
- 決済エラーがある場合に警告表示
- 例：「決済エラー：2件の確認が必要です」

【最近のアップロード（5件）】
| 表示項目 |
|---------|
| アップロード日時 |
| ファイル名 |
| 件数 |
| 金額 |
| ステータス（処理中/完了/エラー） |

【お知らせ】
- QOLC運営からのお知らせを表示

---

### 3.2 明細アップロード

**目的**: 利用明細（レセプト）ファイルをアップロードして決済処理

**フロー**:
```
ステップ0: 施設選択（紐づけ施設が複数の場合のみ）
    ↓
ステップ1: ファイル選択
    ↓
ステップ2: 変換・プレビュー
    ↓
ステップ3: 確認・送信
    ↓
ステップ4: 完了
```

**ステップ0: 施設選択**（紐づけ施設が複数の場合のみ表示）
- 紐づけ済み施設の一覧を表示
- どの施設向けの明細かを選択
- 施設が1つだけの場合はこのステップをスキップ
- 紐づけ施設がない場合は「施設との紐づけがありません。施設にお問い合わせください。」と表示しアップロード不可

**ステップ1: ファイル選択**
- ドラッグ＆ドロップまたはファイル選択
- 対応形式：CSV, Excel (.xlsx, .xls)
- ファイルサイズ上限：10MB
- 使用フォーマット名を表示（加盟店登録時に設定済み）

**ステップ2: 変換・プレビュー**
- アップロードしたファイルをQOLC統一フォーマットに自動変換
- 変換結果のプレビュー表示（テーブル形式）
- エラーがある行はハイライト表示
- エラー内容を明示（「3行目：入居者IDが見つかりません」等）

| 表示カラム |
|-----------|
| 入居者ID |
| 入居者名 |
| サービス日 |
| サービス内容 |
| 金額 |
| 自己負担額 |
| ステータス（OK/エラー） |

**ステップ3: 確認・送信**
- 合計件数、合計金額を表示
- エラー件数がある場合は警告表示
- 「エラー行をスキップして送信」オプション
- 「送信」ボタンで決済処理開始

**ステップ4: 完了**
- アップロード完了メッセージ
- 処理件数、合計金額を表示
- 「履歴を確認」「続けてアップロード」ボタン

---

### 3.3 アップロード履歴

**目的**: 過去のアップロードと決済状況を確認

**一覧画面**:

【検索・フィルター】
- 期間指定（今月/先月/カスタム）
- ステータス（すべて/処理中/完了/エラー）

【一覧テーブル】
| カラム | 説明 |
|--------|------|
| アップロード日時 | アップロードした日時 |
| ファイル名 | 元のファイル名 |
| 件数 | 明細の件数 |
| 合計金額 | 明細の合計金額 |
| 決済完了 | 決済完了した件数/金額 |
| ステータス | 処理中/完了/一部エラー/エラー |
| 操作 | 詳細を見る |

**詳細画面**:

【アップロード情報】
- アップロード日時
- ファイル名
- 全体件数、合計金額
- ステータス

【明細一覧】
| カラム | 説明 |
|--------|------|
| 入居者名 | 入居者の氏名 |
| 施設名 | 所属施設 |
| サービス日 | サービス提供日 |
| サービス内容 | 明細項目 |
| 金額 | 請求金額 |
| 自己負担額 | 入居者負担額 |
| 決済ステータス | 未決済/処理中/完了/エラー |
| 決済日時 | 決済完了日時 |

【エラー明細】
- エラーがある場合、エラー内容を表示
- 「再処理」ボタン（カード情報エラー等の場合）

---

### 3.4 設定

**目的**: アカウント情報・通知設定の管理

【アカウント情報】
- 事業者名（表示のみ）
- 担当者名
- メールアドレス
- パスワード変更

【通知設定】
- アップロード完了通知（メール）：ON/OFF
- 決済完了通知（メール）：ON/OFF
- 決済エラー通知（メール）：ON/OFF

【利用フォーマット】
- 現在設定されているフォーマット名を表示（変更は運営者へ依頼）

## 4. ステータス定義

### アップロードステータス

| ステータス | 説明 | 表示色 |
|-----------|------|--------|
| processing | 処理中（変換・決済処理中） | 青 |
| completed | 完了（全件決済完了） | 緑 |
| partial_error | 一部エラー（一部決済失敗） | 黄 |
| error | エラー（処理失敗） | 赤 |

### 決済ステータス（明細単位）

| ステータス | 説明 | 表示色 |
|-----------|------|--------|
| pending | 未決済（処理待ち） | グレー |
| processing | 処理中 | 青 |
| completed | 決済完了 | 緑 |
| failed | 決済失敗 | 赤 |

## 5. 認証・権限

### 認証方式
- Auth0によるメール + パスワード認証
- MFA推奨

### アクセス権限
- 自社（自分の加盟店）のデータのみ閲覧・操作可能
- 他の加盟店のデータにはアクセス不可

### ユーザー種別（加盟店内）

| 種別 | 権限 |
|------|------|
| 管理者 | 全機能利用可、他ユーザー追加可 |
| スタッフ | アップロード・履歴確認可、設定変更不可 |

## 6. データフロー

### 明細アップロード〜決済完了の流れ

```
1. サービス提供者がCSVをアップロード
    ↓
2. QOLC がフォーマット設定を参照
    ↓
3. 統一フォーマットに変換
    ↓
4. 変換結果をプレビュー表示
    ↓
5. サービス提供者が「送信」
    ↓
6. QOLC が明細を保存
    ↓
7. USEN PSP に決済リクエスト送信
    ↓
8. 決済結果を受信・保存
    ↓
9. サービス提供者に完了通知（メール）
    ↓
10. 入居者・家族に明細通知（LINE/メール）
```

## 8. UIデザイン方針

### カラー
- メインカラー：ブルー系（#2563eb）運営者管理画面と区別
- 成功：緑（#059669）
- エラー：赤（#dc2626）
- 警告：黄（#d97706）

### レイアウト
- シンプルなサイドバー（4項目のみ）
- メインコンテンツは広く取る
- アップロードはステップ形式で分かりやすく

### レスポンシブ
- PC優先（業務利用想定）
- タブレット対応
- スマホは最低限

## 9. エラーハンドリング

### アップロード時のエラー

| エラー | 対処 |
|--------|------|
| ファイル形式エラー | 「CSV または Excel ファイルをアップロードしてください」 |
| ファイルサイズ超過 | 「ファイルサイズは10MB以下にしてください」 |
| 文字コードエラー | 「ファイルの文字コードを確認してください」 |
| 必須項目不足 | 「〇行目：入居者IDが必要です」 |
| 入居者ID不一致 | 「〇行目：該当する入居者が見つかりません」 |

### 決済時のエラー

| エラー | 対処 |
|--------|------|
| カード情報なし | 施設を通じて入居者・家族にカード登録を依頼 |
| カード残高不足 | 施設に連絡 |
| カード期限切れ | 施設に連絡 |
| 通信エラー | 自動リトライ、それでも失敗なら運営者対応 |
