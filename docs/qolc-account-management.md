# QOLC アカウント管理設計書

> ユーザー種別、発行フロー、画面設計

---

## 1. アカウント体系の全体像

QOLCには4種類の利用者がおり、それぞれ異なるポータル（画面）にログインする。

```
┌─────────────────────────────────────────────────────────┐
│                    QOLC システム                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  【運営者管理画面】        【施設ポータル】              │
│   ユニバーサルが            介護施設が                  │
│   ログイン                  ログイン                    │
│      │                         │                       │
│      │                         │                       │
│  【提供者ポータル】        【LINE / Web】               │
│   サービス提供者が          入居者・家族が              │
│   ログイン                  ログイン                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

---

## 2. ユーザー種別と権限

| ユーザー種別 | 説明 | ログイン先 | 主な権限 |
|-------------|------|-----------|---------|
| **運営者** | ユニバーサルのスタッフ | 運営者管理画面 | 全施設・全加盟店の管理 |
| **施設管理者** | 介護施設の管理者・スタッフ | 施設ポータル | 自施設の入居者・家族管理 |
| **提供者** | 加盟店（サービス提供者）の担当者 | 提供者ポータル | 明細アップロード・決済確認 |
| **入居者・家族** | 入居者本人または家族 | LINE / Web | 明細確認・領収書ダウンロード |

## 3. アカウント発行の責任者

### 誰が誰のアカウントを発行するか

```
ユニバーサル（QOLC運営）
  │
  ├─→ 運営者アカウント（自社スタッフ）
  │
  ├─→ 施設管理者アカウント（介護施設登録時）
  │
  └─→ 提供者アカウント（加盟店登録時）


介護施設（施設管理者）
  │
  ├─→ 施設スタッフアカウント（必要に応じて）
  │
  └─→ 入居者・家族アカウント（入居者登録時）
```

### 発行フローの表

| アカウント種別 | 発行者 | 発行タイミング | 管理場所 |
|--------------|--------|--------------|---------|
| 運営者 | ユニバーサル | 手動で追加 | 運営者管理画面 → 運営者管理 |
| 施設管理者 | ユニバーサル | 介護施設登録時 | 運営者管理画面 → 介護施設詳細 |
| 施設スタッフ | 施設管理者 | 必要に応じて | 施設ポータル → スタッフ管理 |
| 提供者 | ユニバーサル | 加盟店登録時 | 運営者管理画面 → 加盟店詳細 |
| 入居者・家族 | 介護施設 | 入居者登録時 | 施設ポータル → 入居者管理 |

## 4. 各アカウントの発行フロー詳細

### 4.1 運営者アカウント

**発行者**: ユニバーサル（既存の運営者）

**フロー**:
1. 運営者管理画面 → 運営者管理 → 新規追加
2. 名前、メールアドレス、ロールを入力
3. 登録ボタンを押下
4. 招待メールが送信される（パスワード設定リンク付き）
5. 新しい運営者がパスワードを設定してログイン可能に

**運営者のロール（権限レベル）**:
| ロール | 権限 |
|--------|------|
| super_admin | 全権限（運営者の追加・削除含む） |
| admin | 通常の管理権限 |
| viewer | 閲覧のみ |

---

### 4.2 施設管理者アカウント

**発行者**: ユニバーサル

**フロー**:
1. 運営者管理画面 → 介護施設管理 → 新規登録
2. 施設情報を入力（施設名、住所、連絡先等）
3. 加盟店情報を入力（事業所番号、振込先等）
4. 管理者アカウント情報を入力
   - 管理者名
   - メールアドレス
5. 登録ボタンを押下
6. セルフィッシュに加盟店として自動登録
7. 施設管理者に招待メール送信
8. 施設管理者がパスワード設定 → 施設ポータルにログイン可能に

**追加の施設ユーザー**:
- 介護施設詳細画面から追加可能
- または施設管理者が施設ポータルから追加可能

---

### 4.3 提供者アカウント

**発行者**: ユニバーサル

**フロー**:
1. 運営者管理画面 → 加盟店管理 → 新規登録
2. 加盟店情報を入力（事業者名、事業所番号、振込先等）
3. 担当者アカウント情報を入力
   - 担当者名
   - メールアドレス
4. 登録ボタンを押下
5. セルフィッシュに加盟店として登録
6. 担当者に招待メール送信
7. 担当者がパスワード設定 → 提供者ポータルにログイン可能に

---

### 4.4 入居者・家族アカウント

**発行者**: 介護施設（施設管理者）

**フロー**:
1. 施設ポータル → 入居者管理 → 新規登録
2. 入居者情報を入力（氏名、生年月日、被保険者番号等）
3. 家族情報を入力
   - 家族名、続柄、連絡先
   - 通知方法を選択（LINE / メール / 郵送）
4. 登録ボタンを押下
5. 通知方法に応じた処理：

| 通知方法 | アカウント発行処理 |
|---------|------------------|
| LINE | QRコード付き案内書を生成 → 施設から家族へ渡す → 家族がLINE友だち追加 → LINEログインで認証 |
| メール | 招待メール送信 → 家族がパスワード設定 → Webでログイン可能に |
| 郵送 | アカウント不要（明細・領収書を紙で郵送） |

## 5. 画面構成

### 5.1 運営者管理画面（ユニバーサル用）

```
運営者管理画面
├── ダッシュボード
│     └── 統計情報（施設数、加盟店数、明細数、決済状況等）
│
├── 介護施設管理
│     ├── 一覧（検索・フィルター付き）
│     ├── 新規登録
│     │     ├── 施設情報
│     │     ├── 加盟店情報（セルフィッシュ連携用）
│     │     └── 管理者アカウント
│     └── 詳細
│           ├── 施設情報（編集可）
│           ├── 施設ユーザー一覧（追加・編集・削除）
│           └── 入居者一覧（閲覧のみ）
│
├── 加盟店管理（介護施設以外）
│     ├── 一覧（検索・フィルター付き）
│     ├── 新規登録
│     │     ├── 加盟店情報
│     │     └── 担当者アカウント
│     └── 詳細
│           ├── 加盟店情報（編集可）
│           └── 提供者ユーザー一覧（追加・編集・削除）
│
├── 運営者管理
│     ├── 一覧
│     ├── 新規追加
│     └── 編集・削除
│
└── 通知管理
      └── 通知履歴・配信状況
```

### 5.2 施設ポータル（介護施設用）

```
施設ポータル

│     ├── 新規登録（入居者情報 + 家族情報）

│     ├── 一覧
│     ├── 新規追加
│     └── 編集・削除
│
├── 明細確認
│     └── 施設全体の明細一覧
│
└── 設定
      └── 領収書発行設定（発行日、自動発行ON/OFF）
```

### 5.3 提供者ポータル（サービス提供者用）

```
提供者ポータル
├── ダッシュボード
│     └── 今月のアップロード数、決済状況等
│
```

```
LINE（LIFF）/ Webアプリ

│

│     └── 月別の明細表示
│
├── 領収書
│     └── 領収書一覧・PDFダウンロード
```

## 6. 認証方式

| ポータル | 認証方式 | 補足 |
|---------|---------|------|
| 運営者管理画面 | Auth0（メール + パスワード + MFA必須） | 高いセキュリティ |
| 施設ポータル | Auth0（メール + パスワード） | MFA推奨 |
| 提供者ポータル | Auth0（メール + パスワード） | MFA推奨 |
| 入居者・家族（LINE） | Auth0 + LINEログイン | LINE IDで認証 |
| 入居者・家族（Web） | Auth0（メール + パスワード） | LINE未利用者向け |

## 7. データモデル（ユーザー関連）

### User（共通ユーザーテーブル）

```typescript
type UserRole = 'admin' | 'facility_staff' | 'provider' | 'family';

interface User {
  id: string;
  email: string;
  name: string;
  role: UserRole;
  
  // ロール別の紐づけ（いずれか1つ）
  adminProfile?: AdminProfile;      // 運営者の場合
  facilityStaffProfile?: FacilityStaffProfile;  // 施設スタッフの場合
  providerProfile?: ProviderProfile; // 提供者の場合
  familyProfile?: FamilyProfile;     // 家族の場合
  
  // 認証関連
  auth0UserId: string;
  lineUserId?: string;
  lastLoginAt?: Date;
  
  status: 'active' | 'invited' | 'disabled';
  createdAt: Date;
  updatedAt: Date;
}
```

### AdminProfile（運営者プロフィール）

```typescript
type AdminRole = 'super_admin' | 'admin' | 'viewer';

interface AdminProfile {
  id: string;
  userId: string;
  adminRole: AdminRole;
}
```

### FacilityStaffProfile（施設スタッフプロフィール）

```typescript
interface FacilityStaffProfile {
  id: string;
  userId: string;
  facilityId: string;
  isAdmin: boolean;  // 施設管理者かどうか
}
```

### ProviderProfile（提供者プロフィール）

```typescript
interface ProviderProfile {
  id: string;
  userId: string;
  merchantId: string;
  isAdmin: boolean;  // 加盟店の管理者かどうか
}
```

### FamilyProfile（家族プロフィール）

```typescript
interface FamilyProfile {
  id: string;
  userId: string;
  residentId: string;
  relationship: string;  // 続柄
  notificationPreference: 'line' | 'email' | 'mail';
}
```
