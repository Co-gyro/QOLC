# QOLC 施設・サービス提供者 紐づけ設計書

> 介護施設とサービス提供者の関係管理

---

## 1. 概要

### 背景

サービス提供者（訪問診療、タクシー等）が明細をアップロードする際、どの施設のどの入居者かを特定する必要がある。そのため、介護施設とサービス提供者の紐づけを管理する仕組みが必要。

### 方針

**介護施設が「利用するサービス提供者」を登録・管理する**

理由：
- 実際の契約は「介護施設 ↔ サービス提供者」間で行われる
- どのサービス提供者を使うかは施設が決める
- 施設の自主管理で運営者の負担を軽減

---

## 2. 紐づけの流れ

```
1. 運営者が加盟店（サービス提供者）をQOLCに登録
   └── この時点では施設との紐づけなし

2. 運営者が介護施設をQOLCに登録
   └── この時点ではサービス提供者との紐づけなし

3. 介護施設が施設ポータルで「利用するサービス提供者」を登録
   └── ここで紐づけが発生（多対多）
   
4. サービス提供者が明細アップロード
   └── 紐づけ済みの施設の入居者のみがマッチング対象
```

---

## 3. 関係の構造

```
介護施設（多）←→（多）サービス提供者

例：
〇〇介護施設
  ├── A訪問診療クリニック
  ├── Bタクシー
  └── C薬局

△△ケアホーム
  ├── A訪問診療クリニック（〇〇と同じ）
  ├── D訪問看護
  └── Eタクシー

A訪問診療クリニック
  ├── 〇〇介護施設
  └── △△ケアホーム
```

---

## 4. データモデル

### FacilityMerchantRelation（施設・サービス提供者紐づけ）

```typescript
interface FacilityMerchantRelation {
  id: string;
  facilityId: string;       // 介護施設ID
  merchantId: string;       // 加盟店（サービス提供者）ID
  status: 'active' | 'inactive';  // 有効/無効
  registeredAt: Date;       // 紐づけ登録日
  registeredBy: string;     // 登録者（施設ユーザーID）
  createdAt: Date;
  updatedAt: Date;
}
```

### インデックス

- facilityId（施設から検索）
- merchantId（サービス提供者から検索）
- facilityId + merchantId（ユニーク制約）

## 5. 画面設計

### 5.1 施設ポータル：サービス提供者管理

```
施設ポータル
├── ダッシュボード
├── 入居者管理
├── サービス提供者管理 ← 追加
│     ├── 利用中のサービス提供者一覧
│     └── サービス提供者を追加
├── 明細確認
└── 設定
```

**利用中のサービス提供者一覧**

| カラム | 説明 |
|--------|------|
| サービス提供者名 | 事業者名 |
| サービス種別 | 訪問診療/タクシー/薬局等 |
| 登録日 | 紐づけした日 |
| ステータス | 有効/無効 |
| 操作 | 無効化/削除 |

**サービス提供者を追加**

1. 登録済みの加盟店一覧から検索
2. サービス提供者名またはサービス種別で絞り込み
3. 選択して「追加」ボタン

---

### 5.2 運営者管理画面：確認機能

**施設詳細画面**

```
介護施設詳細
├── 基本情報
├── 施設ユーザー一覧
├── 入居者一覧（閲覧のみ）
└── 利用サービス提供者一覧 ← 追加（閲覧のみ）
```

**加盟店詳細画面**

```
加盟店詳細
├── 基本情報
├── 提供者ユーザー一覧
└── サービス提供先施設一覧 ← 追加（閲覧のみ）
```

---

### 5.3 サービス提供者ポータル：明細アップロード

**アップロード時の施設選択**

紐づけ済み施設が1つの場合：
- 自動選択（施設選択画面をスキップ）

紐づけ済み施設が複数の場合：
- 最初に施設を選択するステップを追加

```
明細アップロード

ステップ0：施設選択（複数紐づけの場合のみ）
  └── どの施設向けの明細ですか？
      ○ 〇〇介護施設
      ○ △△ケアホーム
      
ステップ1：ファイル選択
ステップ2：プレビュー
ステップ3：確認
ステップ4：完了
```

紐づけ済み施設がない場合：
- 「施設との紐づけがありません。施設にお問い合わせください。」と表示
- アップロード不可

## 6. 運用フロー

### 新しいサービス提供者を利用開始する場合

```
1. 介護施設がサービス提供者と契約（QOLC外の業務）

2. 運営者に連絡「〇〇クリニックを使いたい」

3-A. 既にQOLCに登録済みの場合：
     └── 施設ポータルで「サービス提供者を追加」

3-B. 未登録の場合：
     └── 運営者が加盟店として登録
     └── 施設ポータルで「サービス提供者を追加」

4. サービス提供者が明細をアップロード可能に
```

### サービス提供者の利用を終了する場合

```
1. 施設ポータルで該当サービス提供者を「無効化」または「削除」

2. 無効化した場合：
   └── 過去の明細は閲覧可能、新規アップロード不可

3. 削除した場合：
   └── 紐づけ解除（過去の明細データは残る）
```

---

## 7. 明細マッチングの流れ

```
サービス提供者がCSVをアップロード
    ↓
選択された施設（または唯一の紐づけ施設）を特定
    ↓
CSVの各行について：
    ↓
被保険者番号で、選択施設の入居者を検索
    ↓
┌──────────────────────────────────────┐
│ 見つかった → その入居者の明細として登録  │
│ 見つからない → エラー行としてハイライト   │
└──────────────────────────────────────┘
```

---

## 8. セキュリティ考慮

| ポイント | 対策 |
|---------|------|
| サービス提供者のアクセス範囲 | 紐づけ済み施設の入居者情報のみ閲覧可能 |
| 施設のアクセス範囲 | 自施設に紐づけたサービス提供者のみ表示 |
| 紐づけの変更権限 | 施設管理者のみ（施設スタッフは不可） |
