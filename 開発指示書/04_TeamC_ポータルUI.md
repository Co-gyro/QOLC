## ★ 既存コードについて（重要）

このプロジェクトには Phase 0 で作成済みのコードがあります。**ゼロから作り直すのではなく、既存コードを発展させてください。**

### 既存のモックアップページ（ダミーデータ入り・UIプロトタイプとして使える）
- `src/app/page.tsx` — 運営者ダッシュボード（統計カード、施設一覧、決済一覧）
- `src/app/facility/page.tsx` — 施設ポータル（入居者一覧、アップロード履歴）
- `src/app/provider/page.tsx` — 提供者ポータル（アップロード、施設一覧）
- `src/app/user/page.tsx` — ユーザーポータル（利用明細、カード状態）

### 既存の実装済み機能
- `src/app/admin/csv-tools/` — CSV変換ツール（JCBリネーム、セゾンUR/FM/FI）→ そのまま維持
- `src/app/admin/merchant-application/` — 加盟店申請フォーム（JCB EC/店頭）→ /admin/merchants に統合
- `src/components/ui/` — shadcn/ui コンポーネント（button, card, input, label, tabs, textarea）

### 作業方針
1. 既存の `page.tsx` モックアップのデザイン・レイアウトを参考にする
2. ダミーデータをAPI/DBからの取得に置き換える
3. サブページ（/facility/residents, /facility/statements 等）を新規追加
4. 共通レイアウト（サイドバー、ヘッダー）でラップする
5. 既存の shadcn/ui コンポーネントを活用し、不足分を追加する

## デザインルール（全ポータル共通）

```
【カラー】
- プライマリ: #4C986A（ボタン、アクセント）
- プライマリホバー: #3D7A55
- 背景（薄緑）: #F0F9F4
- ウォームアクセント: #E8913A（注意、ハイライト）
- テキスト: #333333
- ボーダー: #E0DDD8

【タイポグラフィ】
- フォント: Noto Sans JP（日本語）, Inter（英数字）
- 最小フォントサイズ: 14px（高齢者対応、絶対に下回らないこと）
- H1: 36px, H2: 28px, H3: 22px, H4: 18px
- 本文: 16px, キャプション: 14px

【コンポーネント】
- shadcn/ui をベースにカスタマイズ
- ボタン: 最小高さ 44px（タッチ対応）
- フォーム入力: 最小高さ 44px
- テーブル行: 最小高さ 48px
```

## Task C-2: 運営者管理画面（/admin）

```
運営者管理画面のページを実装してください。

【ページ一覧】
/admin/dashboard — ダッシュボード
  - 統計カード: 総施設数、総入居者数、今月の決済件数/金額、エラー件数
  - 直近の決済一覧（最新10件）
  - アラート（カード期限切れ、未処理エラー等）

/admin/facilities — 介護施設管理
  - 施設一覧テーブル（検索、フィルタ、ページネーション）
  - 施設詳細画面（施設情報、所属入居者一覧、加盟店情報）
  - 施設登録フォーム（グループ選択、基本情報、モールコード自動採番）

/admin/merchants — 加盟店管理
  - 加盟店一覧テーブル（ステータスフィルタ: 申請中/承認済/却下）
  - 申請詳細画面（申請内容確認、補足情報入力、モールコード・端末ID採番）
  - 申請書出力ボタン（JCB EC用/店頭用、セゾン用、USEN用の4パターン）
  - 審査結果入力（加盟店番号入力、ステータス更新）

/admin/payments — 決済管理
  - 決済一覧テーブル（ステータスフィルタ、日付範囲、施設フィルタ）
  - 決済詳細（監査ログ表示含む）
  - 未決済一覧（保留中の決済）
  - エラー一覧（失敗した決済）

/admin/csv-tools — CSV変換ツール（Phase 0 から移行）
  - JCBタブ / セゾンタブ
  - ファイルアップロード → プレビュー → 変換 → ダウンロード

/admin/master — マスタ管理
  - サービス種別マスタ
  - アップロードフォーマット管理
  - モールコードプール管理（残数表示、枯渇アラート）
  - 端末識別番号プール管理

【全画面共通】
- ローディング状態の表示
- エラー時のフォールバックUI
- パンくずリスト
- データなし時の空状態表示
```

## Task C-3: 介護施設ポータル（/facility）

```
介護施設ポータルのページを実装してください。

/facility/dashboard — ダッシュボード
  - 入居者数、今月の決済件数/金額、カード未登録者数
  - 直近の決済一覧

/facility/residents — 入居者管理
  - 入居者一覧テーブル（カード登録状態のバッジ表示）
  - 入居者登録フォーム（氏名、被保険者番号、家族アカウント設定）
  - QRコード生成（招待トークン付き）→ 印刷ボタン
  - メール招待送信ボタン
  - 入居者詳細（基本情報、家族アカウント一覧、カード登録状態、決済履歴）

/facility/statements — 明細管理
  - 明細アップロード（CSVファイル → プレビュー → 確認 → 決済実行）
  - アップロード履歴一覧
  - 外部提供者からのアップロード分も表示（自動分類された明細）
  - 月次明細まとめ表示

/facility/payments — 決済状況
  - 決済一覧（入居者別、提供者別でフィルタ可能）
  - 未決済（カード未登録の保留分）一覧

/facility/providers — サービス提供者管理
  - 紐づけ済みの提供者一覧
  - 提供者の追加（登録済み加盟店から選択）/ 解除
```

## Task C-5: ユーザーポータル（/user）

```
ユーザーポータル（入居者・家族向け）のページを実装してください。

★ このポータルは高齢者も使うため、特にフォントサイズ・ボタンサイズに注意
★ 最小フォント 14px、ボタン最小高さ 48px、余白を広めに

/user/home — ホーム
  - お知らせ一覧（未読件数バッジ）
  - 今月のご利用額サマリー
  - カード登録状態（未登録なら大きなバナーで促す）

/user/statements — ご利用明細
  - 月別の利用明細一覧
  - 提供者ごとにグルーピング表示
  - 明細の詳細（サービス名、金額、日付）

/user/receipts — 領収書
  - 月別の領収書一覧
  - PDFダウンロードボタン

/user/card — カード管理
  - 現在のカード情報（下4桁のみ表示: Visa •••• 1234）
  - 支払いオーナーの場合: カード変更ボタン → USEN決済画面へリダイレクト
  - 非オーナーの場合: 閲覧のみ（「変更は支払い担当者のみ可能です」表示）

【デザイン特記】
- 大きめのカード型レイアウト（リスト表示ではなくカードで見やすく）
- 金額は大きな太字で表示
- ステータスはアイコン付きで直感的に
- 操作ボタンは画面下部に固定（モバイル対応）
```
