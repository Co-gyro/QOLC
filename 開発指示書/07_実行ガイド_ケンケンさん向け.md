# 実行ガイド — ケンケンさん向け

**VS Code + Claude Code で5チーム並行開発を進めるための手順書**

---

## 全体の流れ

```
Week 1:  Team A（基盤: A-2〜A-4）を最優先で完了 + Team E（テスト基盤）を並行
         ※ A-1（プロジェクト初期化）は完了済みのためスキップ
Week 2:  Team B（決済）+ Team C（UI）+ Team D（データ処理: D-2〜D-4）が並行開始
         ※ D-1（Phase 0 CSV移行）は完了済みのためスキップ
Week 3:  統合 + Team E がレビュー
Week 4:  全体テスト + 修正 + USENテスト環境での実テスト
```

---

## 準備（最初に1回だけ）

### Step 1: QOLC プロジェクトを使用
既存の QOLC プロジェクトをそのまま使います（新規作成は不要）。
Phase 0（CSV変換、モックアップ等）が完了済みで、51テストが通過しています。

### Step 2: VS Code で QOLC プロジェクトを開く

### Step 3: CLAUDE.md は更新済み
プロジェクトルートの `CLAUDE.md` は Phase 1 対応版に更新済みです。

---

## 各チームの起動方法

### 共通の指示テンプレート

Claude Code の新しいセッション（ウィンドウ）を開いて、以下のように指示します:

```
あなたは QOLC プロジェクトの [Team X] です。
まず CLAUDE.md を読んでプロジェクトを理解してください。
次に 開発指示書/[XX_TeamX_指示書.md] を読んで、タスクを順番に実行してください。

作業ブランチ: feat/[担当]-[機能名]
（例: feat/foundation-db-schema）

ルール:
- 新しいファイルを作ったら必ずテストも作る
- 他チームの担当ディレクトリは変更しない
- 共通ファイルの変更が必要な場合は変更せず、何が必要かを報告する
- セキュリティルール（CLAUDE.md記載）を厳守する
```

---

### Team A: 基盤（★最初に起動）

```
あなたは QOLC プロジェクトの Team A（基盤チーム）です。
まず CLAUDE.md を読んでプロジェクトを理解してください。
次に 開発指示書/02_TeamA_基盤構築.md を読んで、
Task A-2 から順番に実行してください。

※ Task A-1（プロジェクト初期化）は完了済みです。
  ただし、追加パッケージのインストール（@supabase/supabase-js, @supabase/ssr, zod, playwright）
  と .env.local の作成、.gitignore への追記は必要です。

このチームが他のすべてのチームの土台になるので、最優先で完了させてください。

作業ブランチ: feat/foundation-db-schema（Task A-2）
                feat/foundation-auth（Task A-3）
                feat/foundation-types（Task A-4）

完了したら develop にマージして、他チームが使えるようにしてください。
```

## 統合の手順（Week 3）

1. Team A の develop ブランチに全チームが最新をマージ
2. コンフリクトが起きたら、対象チーム同士で解消
3. Team E がセキュリティチェックリストに基づいて全体レビュー
4. E2Eテスト実行
5. 問題があれば修正→再レビュー

---

## トラブル時の対処

| 困ったこと | 対処法 |
|---|---|
| チーム間でコンフリクト | 共通ファイルの変更は Team A に任せる |
| テストが通らない | エラーメッセージをそのまま Claude Code に貼って「直して」 |
| Team A の完了が遅れている | Team B/C/D はモックデータで先に進める（既存モックアップが参考になる） |
| 指示書の仕様が不明確 | QOLC_要件整理_未決定事項一覧.md を参照 |
| セキュリティが不安 | Team E に確認を依頼（チェックリスト使用） |
