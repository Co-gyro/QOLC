# QOLC アカウント管理設計書

> ユーザー種別、発行フロー、画面設計

---

## 1. アカウント体系の全体像

QOLCには4種類の利用者がおり、それぞれ異なるポータル（画面）にログインする。

```
┌─────────────────────────────────────────────────────────┐
│                    QOLC システム                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  【運営者管理画面】        【施設ポータル】              │
│   ユニバーサルが            介護施設が                  │
│   ログイン                  ログイン                    │
│      │                         │                       │
│      │                         │                       │
│  【提供者ポータル】        【利用者ポータル /user】      │
│   サービス提供者が          入居者・家族が              │
│   ログイン                  ログイン                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

---

## 2. ユーザー種別と権限

| ユーザー種別 | 説明 | ログイン先 | 主な権限 |
|-------------|------|-----------|---------|
| **運営者** | ユニバーサルのスタッフ | 運営者管理画面 | 全施設・全加盟店の管理 |
| **施設管理者** | 介護施設の管理者・スタッフ | 施設ポータル | 自施設の入居者・家族管理 |
| **提供者** | 加盟店（サービス提供者）の担当者 | 提供者ポータル | 明細アップロード・決済確認 |
| **入居者・家族** | 入居者本人または家族 | 利用者ポータル（/user） | 明細確認・領収書ダウンロード・カード登録 |

## 3. アカウント発行の責任者

### 誰が誰のアカウントを発行するか

```
ユニバーサル（QOLC運営）
  │
  ├─→ 運営者アカウント（自社スタッフ）
  │
  ├─→ 施設管理者アカウント（介護施設登録時）
  │
  └─→ 提供者アカウント（加盟店登録時）


介護施設（施設管理者）
  │
  ├─→ 施設スタッフアカウント（必要に応じて）
  │
  └─→ 入居者・家族アカウント（入居者登録時）
       ├─→ 入居者本人アカウント（self）
       └─→ 家族アカウント（family）※複数可
```

### 発行フローの表

| アカウント種別 | 発行者 | 発行タイミング | 管理場所 |
|--------------|--------|--------------|---------|
| 運営者 | ユニバーサル | 手動で追加 | 運営者管理画面 → 運営者管理 |
| 施設管理者 | ユニバーサル | 介護施設登録時 | 運営者管理画面 → 介護施設詳細 |
| 施設スタッフ | 施設管理者 | 必要に応じて | 施設ポータル → スタッフ管理 |
| 提供者 | ユニバーサル | 加盟店登録時 | 運営者管理画面 → 加盟店詳細 |
| 入居者・家族 | 介護施設 | 入居者登録時 | 施設ポータル → 入居者管理 |

## 4. 各アカウントの発行フロー詳細

### 4.1 運営者アカウント

**発行者**: ユニバーサル（既存の運営者）

**フロー**:
1. 運営者管理画面 → 運営者管理 → 新規追加
2. 名前、メールアドレス、ロールを入力
3. 登録ボタンを押下
4. 招待メールが送信される（パスワード設定リンク付き）
5. 新しい運営者がパスワードを設定してログイン可能に

**運営者のロール（権限レベル）**:
| ロール | 権限 |
|--------|------|
| super_admin | 全権限（運営者の追加・削除含む） |
| admin | 通常の管理権限 |
| viewer | 閲覧のみ |

---

### 4.2 施設管理者アカウント

**発行者**: ユニバーサル

**フロー**:
1. 運営者管理画面 → 介護施設管理 → 新規登録
2. 施設情報を入力（施設名、住所、連絡先等）
3. 加盟店情報を入力（事業所番号、振込先等）
4. 管理者アカウント情報を入力
   - 管理者名
   - メールアドレス
5. 登録ボタンを押下
6. セルフィッシュに加盟店として自動登録
7. 施設管理者に招待メール送信
8. 施設管理者がパスワード設定 → 施設ポータルにログイン可能に

**追加の施設ユーザー**:
- 介護施設詳細画面から追加可能
- または施設管理者が施設ポータルから追加可能

---

### 4.3 提供者アカウント

**発行者**: ユニバーサル

**フロー**:
1. 運営者管理画面 → 加盟店管理 → 新規登録
2. 加盟店情報を入力（事業者名、事業所番号、振込先等）
3. 担当者アカウント情報を入力
   - 担当者名
   - メールアドレス
4. 登録ボタンを押下
5. セルフィッシュに加盟店として登録
6. 担当者に招待メール送信
7. 担当者がパスワード設定 → 提供者ポータルにログイン可能に

---

### 4.4 入居者・家族アカウント（アカウント登録）

**発行者**: 介護施設（施設管理者）

**フロー**:
1. 施設ポータル → 入居者管理 → 新規登録
2. ステップ1: 入居者情報を入力（氏名、生年月日、被保険者番号等）
3. ステップ2: アカウント登録
   - アカウント種別を選択（入居者本人 / 家族）
   - **入居者本人アカウント**: 通知方法選択（LINE/メール/郵送）、決済担当チェック
   - **家族アカウント**: 氏名、続柄、電話番号、メール、通知方法、明細通知受取、決済担当チェック
   - 決済担当は1名のみ（排他選択：チェックすると他のアカウントのチェックが外れる）
   - 「アカウントを追加」ボタンで複数アカウント登録可能
4. ステップ3: 確認画面で入力内容を確認
5. 登録ボタンを押下
6. 完了画面: アカウント種別に応じた案内表示

**完了画面の表示**:

| アカウント種別 | 通知方法 | 表示内容 |
|--------------|---------|---------|
| 入居者本人 | LINE | 👤 QRコード付き案内書を印刷 →「入居者ご本人様にお渡しください」 |
| 入居者本人 | メール | 👤 招待メール送信済み + メールアドレス表示 |
| 家族 | LINE | 👥 QRコード付き案内書を印刷 →「ご家族様にお渡しください」 |
| 家族 | メール | 👥 招待メール送信済み + メールアドレス表示 |
| いずれか | 郵送 | アカウント不要（明細・領収書を紙で郵送） |

**カード登録について**:
- カード登録は施設では行わない
- 入居者ご本人様またはご家族様が利用者ポータル（/user/card）から登録する
- 決済担当者のみがカード登録可能

## 5. 画面構成

### 5.1 運営者管理画面（ユニバーサル用）

```
運営者管理画面
├── ダッシュボード
│     └── 統計情報（施設数、加盟店数、明細数、決済状況等）
│
├── 介護施設管理
│     ├── 一覧（検索・フィルター付き）
│     ├── 新規登録
│     │     ├── 施設情報
│     │     ├── 加盟店情報（セルフィッシュ連携用）
│     │     └── 管理者アカウント
│     └── 詳細
│           ├── 施設情報（編集可）
│           ├── 施設ユーザー一覧（追加・編集・削除）
│           └── 入居者一覧（閲覧のみ）
│
├── 加盟店管理（介護施設以外）
│     ├── 一覧（検索・フィルター付き）
│     ├── 新規登録
│     │     ├── 加盟店情報
│     │     └── 担当者アカウント
│     └── 詳細
│           ├── 加盟店情報（編集可）
│           └── 提供者ユーザー一覧（追加・編集・削除）
│
├── 運営者管理
│     ├── 一覧
│     ├── 新規追加
│     └── 編集・削除
│
└── 通知管理
      └── 通知履歴・配信状況
```

### 5.2 施設ポータル（介護施設用）

```
施設ポータル
├── ダッシュボード
│     └── 入居者数、今月の明細数、通知状況等
│
├── 入居者管理
│     ├── 一覧（アカウント数表示：「3名（本人 + 家族2名）」）
│     ├── 新規登録（入居者情報 + アカウント登録）
│     └── 詳細
│           ├── 入居者情報（編集可）
│           ├── アカウント一覧（追加・編集・削除、👤本人/👥家族）
│           ├── 明細履歴
│           └── 領収書履歴
│
├── スタッフ管理（オプション）
│     ├── 一覧
│     ├── 新規追加
│     └── 編集・削除
│
├── 明細確認
│     └── 施設全体の明細一覧
│
└── 設定
      └── 領収書発行設定（発行日、自動発行ON/OFF）
```

### 5.3 提供者ポータル（サービス提供者用）

```
提供者ポータル
├── ダッシュボード
│     └── 今月のアップロード数、決済状況等
│
├── 明細アップロード
│     └── ファイルアップロード（CSV対応）
│
├── アップロード履歴
│     └── 過去のアップロード一覧・詳細
│
└── 決済状況
      └── 決済完了・未決済の一覧
```

### 5.4 利用者ポータル（/user）

```
利用者ポータル（/user）
├── ホーム（/user）
│     └── アカウント種別に応じた挨拶、お知らせ、最新の明細
│     └── カード未登録バナー（決済担当者のみ）
│
├── 明細（/user/statements）
│     ├── 明細一覧（月別）
│     └── 明細詳細（/user/statements/[id]）
│
├── 領収書（/user/receipts）
│     └── 領収書一覧・PDFダウンロード
│
├── カード（/user/card）
│     └── カード登録/変更（決済担当者のみ）
│     └── 非決済担当者には「決済担当ではありません」表示
│
└── 設定（/user/settings）
      ├── アカウント情報（種別、続柄、決済担当）
      └── 通知設定
```

## 6. 認証方式

| ポータル | 認証方式 | 補足 |
|---------|---------|------|
| 運営者管理画面 | Auth0（メール + パスワード + MFA必須） | 高いセキュリティ |
| 施設ポータル | Auth0（メール + パスワード） | MFA推奨 |
| 提供者ポータル | Auth0（メール + パスワード） | MFA推奨 |
| 利用者ポータル（LINE） | Auth0 + LINEログイン | LINE IDで認証 |
| 利用者ポータル（Web） | Auth0（メール + パスワード） | LINE未利用者向け |

## 7. データモデル（ユーザー関連）

### User（共通ユーザーテーブル）

```typescript
type UserRole = 'admin' | 'facility_staff' | 'provider' | 'user';

interface User {
  id: string;
  email: string;
  name: string;
  role: UserRole;
  
  // ロール別の紐づけ（いずれか1つ）
  adminProfile?: AdminProfile;      // 運営者の場合
  facilityStaffProfile?: FacilityStaffProfile;  // 施設スタッフの場合
  providerProfile?: ProviderProfile; // 提供者の場合
  accountProfile?: AccountProfile;   // 入居者本人・家族の場合
  
  // 認証関連
  auth0UserId: string;
  lineUserId?: string;
  lastLoginAt?: Date;
  
  status: 'active' | 'invited' | 'disabled';
  createdAt: Date;
  updatedAt: Date;
}
```

### AdminProfile（運営者プロフィール）

```typescript
type AdminRole = 'super_admin' | 'admin' | 'viewer';

interface AdminProfile {
  id: string;
  userId: string;
  adminRole: AdminRole;
}
```

### FacilityStaffProfile（施設スタッフプロフィール）

```typescript
interface FacilityStaffProfile {
  id: string;
  userId: string;
  facilityId: string;
  isAdmin: boolean;  // 施設管理者かどうか
}
```

### ProviderProfile（提供者プロフィール）

```typescript
interface ProviderProfile {
  id: string;
  userId: string;
  merchantId: string;
  isAdmin: boolean;  // 加盟店の管理者かどうか
}
```

### AccountProfile（アカウントプロフィール）

```typescript
type AccountType = 'self' | 'family';

interface AccountProfile {
  id: string;
  userId: string;
  residentId: string;
  accountType: AccountType;       // 入居者本人 or 家族
  relationship?: string;          // 続柄（家族の場合）
  notifyMethod: 'line' | 'email' | 'mail';
  receiveNotify: boolean;         // 明細通知を受け取るか
  isBillingPerson: boolean;       // 決済担当者（1名のみ）
}
```

## 8. まとめ

### アカウント発行の責任

| 発行されるアカウント | 発行者 |
|-------------------|--------|
| 運営者 | ユニバーサル |
| 施設管理者 | ユニバーサル |
| 施設スタッフ | 施設管理者 |
| 提供者 | ユニバーサル |

### 画面の分離

| 画面 | 対象ユーザー | 主な目的 |
|------|------------|---------|
| 運営者管理画面 | ユニバーサル | 全体管理 |
| 施設ポータル | 介護施設 | 入居者管理 |
| 提供者ポータル | サービス提供者 | 明細アップロード |
| 利用者ポータル（/user） | 入居者・家族 | 明細確認・領収書取得・カード登録 |
